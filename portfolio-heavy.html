Machine Anxiety

2026 — Heavy point-cloud WebGL revision + real-time data-driven sound

Machine Anxiety is a data-driven WebGL work that translates economic volatility into the destabilisation of a scanned domestic room.

This revised version develops the original black-and-white point-cloud system into a heavier colour scan of a student room. The work uses historical GBP/CNY exchange-rate data to activate spatial pressure, point-cloud breathing, delayed erosion, automated camera movement, and a live Web Audio system.

The project treats anxiety not as an internal emotion, but as a computational pressure acting on private space. Financial fluctuation enters the room as a force: first as subtle breathing deformation, then as accumulated particle dispersal, and finally as a partial loss of ceiling, wall, and object stability.

Viewing Experience

The work opens with an entry screen. After entering, the data timeline begins and the room performs as a responsive system. The camera starts outside the scanned room, slowly moves around the side of the architectural volume, approaches the window/bed area, and enters the interior. After the first major data pressure event, the view rises toward an overhead perspective, then returns to a side view to observe the later instability.

The room does not simply visualise a graph. Instead, exchange-rate movement becomes a behavioural condition inside the scan. Low pressure produces subtle breathing and spatial vibration. Peaks produce object-level dispersal and delayed structural erosion. Later data events accumulate rather than reset, so the room carries the memory of previous volatility.

System Logic

Data

Historical GBP/CNY exchange-rate values from September 2024 to April 2026 drive the timeline.

Volatility

Rate change and shock are translated into deformation intensity, particle dispersal, sound density, and system pressure.

Room Behaviour

The space first breathes and compresses, then gradually loses coherence through accumulated point-cloud instability.

Camera

A designed automatic camera path moves from exterior observation into the room, then returns to structural distance after the first burst.

Model Revision

The revised model uses a cleaned Scaniverse room scan exported as a PLY point cloud. Earlier tests showed that the raw point cloud appeared too sparse and too mechanically regular. The current version adds a controlled irregularisation layer inside the WebGL system: points are subtly jittered, selectively duplicated near their original positions, and given slight colour variation. This makes the scan feel less like a grid and more like an unstable spatial memory.

The current model file is models/scaniverse_room_clean_585k.ply. The system label is simplified to DORM so the loading state reads as part of the work rather than as a technical filename.

Sound System

The revised version adds a real-time Web Audio system driven by the same exchange-rate data as the visuals. The sound is intentionally restrained and bio-digital rather than cinematic. It behaves like unstable vital signs inside a scanned domestic environment.

  • Low-frequency pressure oscillator for slow breathing tension
  • Irregular sub-bass pulse for non-musical heartbeat-like pressure
  • Short high-frequency data clicks whose pitch follows trend direction
  • Filtered noise for room air, particle activity, and screen hum
  • Brief overload glitches during extreme volatility

Interaction

The public version is designed to play automatically after the viewer enters the work. Mouse or keyboard movement can interrupt the automatic view and allow manual exploration. The debug controls remain available for development, but the main viewing logic is structured around a guided cinematic path.

Export

The system includes an offline frame-sequence export workflow for video production. The export target is a 1920 × 1080 JPG sequence at 30 fps, designed for assembly and final grading in Adobe Premiere Pro. The on-screen data panel is composited during export so the visual state, data state, and sound logic can remain aligned in documentation and final video production.

Technical Notes

  • Custom Three.js / WebGL point-cloud system
  • Colour PLY point-cloud room scan, cleaned and reprocessed for browser use
  • Deterministic point-cloud irregularisation for denser, less grid-like visual behaviour
  • Historical GBP/CNY exchange-rate dataset stored as JSON
  • Data mappings for breathing deformation, delayed erosion, particle dispersal, camera pressure, and sound
  • Real-time Web Audio API synthesis
  • Automatic camera choreography with manual override
  • CCapture.js frame-sequence export for video production
  • Separate GitHub/Vercel project to protect the original submitted version

Version Management

Future Development

The current revision establishes a reusable structure for future versions: different room scans, higher-density local rendering, alternative datasets, more complex deformation systems, and future links to software such as TouchDesigner or Unreal Engine. The key principle is to preserve the relationship between external data pressure and intimate spatial instability.
